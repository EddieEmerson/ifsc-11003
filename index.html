    • Project Reflection: Mars Rover Site

    For this project revision, I focused on improving the technical structure and accessibility of my Mars Rover micro-site. Based on feedback, I ensured that the navigation was consistent across all three pages and that the CSS Grid layout was responsive for mobile users. I also corrected the footer to include the required NASA disclaimer and updated the contact information to my student email.

    What I did well: I successfully implemented a clean, semantic HTML5 structure and used CSS Grid to create a professional-looking comparison table and rover overview. What I would do differently: If I were to start over, I would spend more time optimizing custom imagery from the NASA archives to create a more immersive visual experience.

    Feedback Report: Peer feedback suggested that the initial draft lacked a way to return to the home page from the FAQ section. I addressed this by duplicating the navigation header across all pages. I also adjusted the color contrast in the CSS to ensure the text meets accessibility standards for space-themed designs.

    Thoughts on Progressive Enhancement

    In my own words, progressive enhancement is a "content-first" approach to web development. It means building a website starting with a solid foundation of plain HTML so that the core information and functionality work for everyone, regardless of their internet speed or browser age. Once that basic version is stable, you "enhance" it with layers of CSS for styling and JavaScript for interactivity.

    This strategy is vital in modern web development because the web is accessed on an unpredictable range of devices—from high-end desktops to budget smartphones on slow 3G networks. By using progressive enhancement, we ensure that a JavaScript error or a slow connection doesn't "break" the site. It improves accessibility for screen readers, boosts SEO because search engines can easily read the HTML, and creates a more resilient user experience.
